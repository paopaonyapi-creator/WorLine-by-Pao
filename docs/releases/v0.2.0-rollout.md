# v0.2.0 Rollout & Rollback Plan

This document acts as the live playbook when pressing "Deploy" for `v0.2.0`. It focuses strictly on immediate production observations and definitive rollback criteria.

## Rollout Sequence

### 1. Before Deploy
- Ensure `pnpm test:e2e` and the [Pre-Release Checklist](../pre-release-checklist.md) have 100% passed.
- Confirm any manual visual checks inside the [QA Matrix](../manual-qa-matrix.md) are stamped.
- Double-check your Vercel/Railway instance possesses the matching `ADMIN_EMAILS` variables required.

### 2. During Deploy
- Monitor the Railway build logs actively.
- Watch for missing dependency warnings (e.g., forgotten `pdf-lib` installs).

### 3. First 5 Minutes After Deploy (Safety Bounds)
Immediately open the production URL and execute:
- Hit `/api/health` → Must return `200 OK`.
- Access `/app` while logged out → Must bounce to `/login` smoothly.
- Access `/app` while logged in → Must load Dashboard successfully (Ensure no `/misconfigured` redirects).
- Open `app/settings/billing` → Must render active plan limits securely.

### 4. First 30 Minutes After Deploy (Active Monitoring)
- Monitor Stripe Dashboard Webhooks internally for `500` failures.
- Check Supabase Dashboard → Auth Logs for any spike in authentication rejection.
- Attempt a raw PDF export manual test locally via the live URL.

## Rollback Decision Tree

In a solo-maintainer environment, executing a hard rollback (reverting the deployment commit backward in Railway) vs. pushing a fresh hotfix forward depends purely on the severity of the structural break. Use the following logic conditionally:

* **CRITICAL 🛑:** If `/api/health` fails or 500s.
  → **ACTION:** Immediate Rollback.
* **CRITICAL 🛑:** If `/app` aggressively forces all users to `/misconfigured`.
  → **ACTION:** Immediate Environmental Fix (check `.env` values in Railway) OR Rollback within 5 minutes.
* **CRITICAL 🛑:** If `/api/checkout` triggers `500` errors actively blocking upgrades.
  → **ACTION:** Pause release marketing / Execute an immediate Hotfix blocking Billing routes or Rollback immediately if Stripe Keys leaked.
* **HIGH ⚠️:** If `/admin` blocks all users despite correct arrays.
  → **ACTION:** Do not rollback immediately. Check Supabase connection limits and explicitly verify `.env.local` syntax before rolling a hotfix forward.
* **LOW 🟢:** If non-critical UI padding, color scaling, or localized non-breaking text issues appear.
  → **ACTION:** Continue normally and bundle the fix into the next standard milestone hotfix!
