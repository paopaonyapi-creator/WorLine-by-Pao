# WorLine v0.2.0 Release Notes

**Date:** 2026-03-28

## Highlights & Final Hardening Summary
This release marks a massive functional stabilization pass across the entirety of WorLine. We focused intensely on eliminating silent failures and hardening the application to standard "Fail-Closed" behavior.

> **Deploying this version?** Consult the strict [Rollout & Rollback Plan](v0.2.0-rollout.md) checklist tracking hotfix decision logic!

### Security / Hardening
- **Fail-Closed Auth Behavior:** Middleware now strictly verifies identity and layout context. Missing `.env` layers or unauthenticated requests route safely `/login` gracefully without throwing raw 500 crashes.
- **Admin Allowlist Protection:** The `/admin` domain is firmly guarded via server-side checks traversing the `ADMIN_EMAILS` array directly securely instead of depending purely on visual client-side layers.

### Billing / Stripe
- **Server-Bound Checkout Identity:** Calls to `/api/checkout` look up the executing user securely off `supabase.auth.getUser()`, completely ignoring any falsified client payload manipulation.
- **Webhook Period Persistence:** Stripe `checkout.session.completed` events correctly extrapolate exact subscription cycle dates (start/end) through webhook parsers instead of forcing the dashboard to manually crawl Stripe's API continuously. 
- **Checkout Proxies:** E2E validation ensures the user pricing upgrade route reliably hits `/signup` if logged out and successfully intercepts dynamic Stripe Session payloads if logged in. 

### Editor / UX / Responsive
- **Mobile Stabilization:** App shell components and Workspace navigation drawers respond to parallel Playwright `'Mobile Chrome'` contexts, firmly asserting `<SheetTrigger>` elements and drawing palettes overlap flawlessly.
- **Save/Export Hooks:** Core canvas rendering boundaries successfully persist to the DB smoothly triggering confirmation Toasts and executing synchronous `/api` endpoints without blocking standard NextJS `useEffect` behaviors.

### Testing / CI
- Added dedicated Playwright smoke configurations explicitly separating `.env` seed logic via `requireUserSeed()`/`requireAdminSeed()`.
- Added localized `mockEnv()` vitest stubs isolating raw backend pipeline paths from muddying global state cleanly!

### Ops / Documentation
- Standardized Solo-Maintainer documentation: `Production Runbook`, `Pre-Release Checklist`, `QA Matrix`, and a definitive `Local Test Seed Workflow`.

## Known Limitations
- E2E scripts stop explicitly before executing raw test credit-card submissions onto Stripe environments to aggressively prevent rate-limits and proxy flags via Automated GH Actions.
- While Konva saves successfully to the database structurally, physical PDF/PNG Export validations visually continue to rely on manual developer eyes through the QA Matrix tightly due to headless binary timeout limits efficiently.

## Upgrade / Env Notes
Please ensure the following environment variables are assigned locally inside `.env.local` or securely in Railway:
- `PLAYWRIGHT_TEST_USER_EMAIL` / `PLAYWRIGHT_TEST_USER_PASSWORD`
- `PLAYWRIGHT_TEST_ADMIN_EMAIL` / `PLAYWRIGHT_TEST_ADMIN_PASSWORD`
- `ADMIN_EMAILS`
