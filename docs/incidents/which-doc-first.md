# Which Incident Doc Do I Open First? 🧭

*Overwhelmed? Choose the exact sentence that matches what you know right now.*

**"I was just paged and I don't know the scope yet."**
👉 Start at [The First 10 Minutes](first-10-minutes.md).

**"I only need the most common 5 cases."**
👉 Start at [Top Recurring Incidents](top-recurring.md).

**"I know exactly what is broken."**
👉 Find it in the [Incident Matrix](matrix.md) or the [Main Index](index.md).

**"I know who it is affecting."**
- It's breaking basic user features: [Production Incidents](production-incidents.md)
- It's breaking maintainer tools: [Workflow Incidents](workflow-incidents.md)

**"I know the domain."**
- Payments are weird: [Billing Hub](billing-hub.md)
- Identity is broken: [Auth & Access Hub](auth-access-hub.md)
- The UI is misbehaving: [Editor & UI Hub](editor-ui-hub.md)

**"I need to fix the data right now."**
- Before you edit a row: [Manual DB Intervention Safety](manual-db-intervention-safety.md)
- Before you unblock a user: [Temporary Unblock Safety](temp-unblock-safety.md)

**"I need to talk to users right now."**
- For a status page update: [Outage Update Template](outage-update-template.md)
- For a single user: [Direct User Reply Templates](direct-user-reply-templates.md)

**"I need to know what these emojis mean."**
👉 Open the [Tag Legend](tag-legend.md).
